# Val Ark

**Created by Matthew Valancy**

A comprehensive offline AI toolkit for NVIDIA Jetson, macOS Apple Silicon, and Windows. Downloads ~500GB of models and ~4GB of inference engine binaries across all platforms.

## Components

| File | Purpose |
|------|---------|
| `download-all-models.sh` | Downloads ~500GB of AI models (GGUF, ONNX, SafeTensors) |
| `download-all-tools.sh` | Downloads ~4GB of inference engine binaries + source |
| `index.html` | Static web inventory page - browse models, copy commands |
| `monitor.sh` | Real-time download progress monitoring |
| `retry-failed.sh` | Retry helper for failed model downloads |
| `MODEL_INVENTORY.md` | Detailed model inventory with sizes |

**Generated by `download-all-tools.sh`:**

| File | Location | Purpose |
|------|----------|---------|
| `build-from-source.sh` | `~/models/tools/` | Build llama/whisper/sd.cpp with CUDA on Jetson |
| `build-macos.sh` | `~/models/tools/` | Build whisper.cpp with Metal on macOS |

## System Requirements

- **Hardware**: NVIDIA Jetson (AGX Orin, Orin NX, etc.), macOS Apple Silicon, or Windows x64
- **Storage**: ~600GB free for models + tools
- **Software**: `wget`, `curl`, `git`, `unzip`; HuggingFace CLI (`hf`) for gated models
- **Build (Jetson)**: `cmake`, `ninja-build`, CUDA toolkit (for source builds)
- **Build (macOS)**: Xcode Command Line Tools, `cmake`
- **Network**: Stable internet connection

## Quick Start

```bash
# Pre-validate all URLs (no download, just HEAD checks):
./download-all-models.sh validate

# Download everything (~500GB):
./download-all-models.sh all

# Download by category:
./download-all-models.sh llm      # LLM models (~300GB)
./download-all-models.sh tts      # Text-to-Speech (~15GB)
./download-all-models.sh stt      # Speech-to-Text (~20GB)
./download-all-models.sh vision   # Vision Language Models (~40GB)
./download-all-models.sh image    # Image Generation (~40GB)
./download-all-models.sh nvidia   # NVIDIA Special (~10GB)
./download-all-models.sh extra    # Additional quality models (~75GB)

# Run overnight with nohup:
nohup ./download-all-models.sh all > ~/models/logs/nohup_output.log 2>&1 &

# Monitor progress (separate terminal):
./monitor.sh

# Retry any failures:
./retry-failed.sh
```

### Quick Start (Tools)

```bash

# Download individual tools:

./download-all-tools.sh piper     # Piper TTS

./download-all-tools.sh ffmpeg    # FFmpeg

cd ~/models/tools && ./build-from-source.sh
```

## Web Inventory Page

Open `index.html` in any browser for an interactive model inventory:

```bash
# Locally:
open index.html                    # macOS
xdg-open index.html                # Linux
start index.html                   # Windows

# Or serve over network:
python3 -m http.server 8888
# Then visit http://<ip>:8888/index.html
```

**Features:**
- Platform selector (Jetson / macOS / Windows) - adjusts all paths and commands
- 58 model cards grouped by category (LLM, TTS, STT, Vision, ImageGen, NVIDIA)
- Each card shows: name, size, quantization, format, tool, and a copy-able run command
- Quick Start guides for 5 use cases (Chat, Transcribe, TTS, Image Gen, Vision)
- Search/filter across all models
- "TOP PICK" badges on recommended models
- Storage breakdown visualization
- Fully offline, no external dependencies
- Remembers platform selection (localStorage)

## Running Models (Examples)

### Chat with an LLM

```bash
# Jetson (after build-from-source.sh):
~/models/tools/llama.cpp/linux-arm64/llama-server \
    -m ~/models/llm/llama-3.2-1b/Llama-3.2-1B-Instruct-Q8_0.gguf \
    -ngl 999 -c 2048 --port 8080

# macOS:
~/models/tools/llama.cpp/macos-arm64/llama-server \
    -m ~/models/llm/qwen2.5-32b/Qwen2.5-32B-Instruct-Q4_K_M.gguf \
    -ngl 999 -c 4096 --port 8080

# Then query the API:
curl http://localhost:8080/v1/chat/completions \
    -d '{"messages":[{"role":"user","content":"Hello!"}]}'
```

### Transcribe Audio

```bash
    -m ~/models/stt/whisper-ggml/ggml-large-v3-turbo-q5_0.bin \
    -f recording.wav

# macOS (after build-macos.sh):

    -m ~/models/stt/whisper-ggml/ggml-large-v3-turbo-q5_0.bin \
```

### Text to Speech


## Error Handling

Both download scripts are designed to **never abort**:
- Individual failures don't stop other downloads
- HTTP 404 and 401/403 errors are detected early and not retried
- Transient failures use linear backoff retries (5 attempts)
- Empty/zero-byte partial files are automatically cleaned up on re-run
- All failures are logged with timestamps to the logs directory
- Progress counters track success/failed/skipped counts in real-time
- Resume-capable: re-running skips already-downloaded files
- Tools use `wget -c` (resume), models use GitHub API for dynamic version resolution

### Category 1: LLM Models (~300GB)

- **Nemotron-3-Nano-30B-A3B**: MoE architecture, only 3.2B params active per token despite 30B total. Fast and efficient.

## Usage with llama.cpp

```bash
# Build llama.cpp with CUDA:
cd ~/Code/llama.cpp
cmake -B build -DGGML_CUDA=ON && cmake --build build -j$(nproc)

# Run a GGUF model (API server):
./build/bin/llama-server \
    -m ~/models/llm/nemotron-3-nano-30b/Nemotron-3-Nano-30B-A3B-Q4_K_M.gguf \
    -ngl 999 -c 4096 --port 8080

# Run whisper:
./build/bin/whisper-cli \
    -m ~/models/stt/whisper-ggml/ggml-large-v3-turbo-q5_0.bin \
    -f audio.wav

# Run with vision (LLaVA):
./build/bin/llama-llava-cli \
    -m ~/models/vlm/llava-1.6-mistral/llava-v1.6-mistral-7b.Q4_K_M.gguf \
    --mmproj ~/models/vlm/llava-1.6-mistral/mmproj-model-f16.gguf \
    -ngl 999 --image photo.jpg -p "Describe this image"
```

## Performance Tips for NVIDIA Jetson

1. **Power mode**: `sudo nvpmodel -m 0 && sudo jetson_clocks`
2. **GPU offload**: Always use `-ngl 999` to offload all layers
3. **Flash attention**: Enabled by default (`--flash-attn auto`), gives 10-15% speed boost
4. **Context size**: Start with `-c 4096`, increase if needed (uses more VRAM)
5. **MoE models**: Nemotron-3-Nano-30B works well since only 3.2B active params
6. **Batch size**: For throughput, increase `-np` (parallel slots)
7. **Memory**: Jetson uses unified memory (GPU and CPU share RAM) - leave headroom for KV cache when selecting model sizes

## Troubleshooting

**Gated models (401/403)**:
```bash
# Login to HuggingFace:
~/.local/bin/hf login
# Then accept model licenses on huggingface.co and re-run
```

**GitHub rate limits (tool downloads)**:
```bash
# Set a personal access token:
export GITHUB_TOKEN=ghp_your_token_here
./download-all-tools.sh all
```

**Disk space**: Monitor with `df -h ~/models`. Both scripts log available space at start.

**Partial/corrupt downloads**: Re-run the script - it skips completed files and retries failures.

**CUDA build fails**: Ensure CUDA toolkit is installed:
```bash
nvcc --version       # Should show CUDA compiler
dpkg -l | grep cuda  # Check installed CUDA packages
```

**macOS build fails**: Ensure Xcode tools are installed:
```bash
xcode-select --install
cmake --version
```

**Piper model config missing**: Each `.onnx` voice model needs a `.onnx.json` config file alongside it. Download from Hugging Face:
```bash
# Example for en_US-lessac-medium:
curl -L -o models/tts/piper-voices/en_US-lessac-medium.onnx.json \
    "https://huggingface.co/rhasspy/piper-voices/resolve/main/en/en_US/lessac/medium/en_US-lessac-medium.onnx.json"
```

**Validate URLs before downloading**:
```bash
./download-all-tools.sh validate   # Tools (uses range requests)
./download-all-models.sh validate  # Models (uses HEAD requests)
```
